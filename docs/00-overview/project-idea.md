## 🏆 프로젝트: GameCup (가제: 인생 게임 월드컵)

### 1. 프로젝트 아이디어 개요서 (Updated)

- **프로젝트 이름:** **GameCup: World Cup of My Life Games**
    
- **목적:** `my9games` 라는 사이트에서 영감을 받아 기존 스타일의 정적인 리스트업을 넘어, 사용자가 직접 **'토너먼트'** 를 거치며 자신의 진정한 게임 취향을 확인하고 그 과정을 지인들과 공유하게 함.
    
- **타겟:** 자신의 게임 취향을 확고히 하고 싶은 게이머, 친구들과 게임 취향으로 토론하기 좋아하는 커뮤니티 유저.
    
- **해결하려는 문제:** "좋아하는 게임이 너무 많은데 뭘 골라야 하지?" → 대결을 통해 본능적인 우선순위 도출.
    - "설문은 지루해" → 게임 요소를 가미해 설문 자체를 하나의 콘텐츠로 만듦.

---

## 2. 설문 시스템 기능 아이디어 (Function Idea List)

### 🛠 핵심 기능: 월드컵 빌더 (The Builder)

- **유동적 후보 선택:** 유저가 RAWG API를 통해 **원하는 만큼(최소 4강, 8강, 16강, 32강 등)** 게임을 검색해 후보 리스트에 추가합니다.
- **후보군 검증:** 검색 시 중복 선택 방지 및 이미지 로딩 상태 확인.
- **토너먼트 알고리즘:** 선택된 후보들을 무작위로 섞어 대진표를 생성합니다.

### 🎮 메인 이벤트: 월드컵 진행 (The Tournament)

- **1:1 대결 UI:** 두 게임의 포스터를 양옆에 배치하고 클릭 시 다음 라운드 진출.
- **애니메이션 효과:** 선택 시 이미지가 커지거나 반대편 이미지가 사라지는 등의 연출 (손맛 강조).
- **포기/다시 시작:** 도중에 마음이 바뀌면 언제든 초기화 가능.

### 📝 심화 설문: 우승작 인터뷰 (The Interview)

- **조건부 질문:** 최종 우승 게임이 결정된 직후에만 나타나는 설문 섹션.
    
    - _"이 게임을 처음 접한 시기는?"_
    - _"이 게임의 인생 스크린샷이나 한 줄 평은?"_
    - _"이 게임을 한마디로 정의한다면? (예: '내 눈물의 근원', '우정 파괴자')"_

### 📊 결과 공유 및 통계 (The Finale)

- **결과 카드 생성:** 최종 우승작 포스터를 메인으로, 4강까지 진출했던 게임들을 서브로 보여주는 **커스텀 이미지 카드** 생성.
- **SNS 공유:** 생성된 카드 이미지를 다운로드하거나 URL로 공유.
- **전체 통계:** "이 월드컵에서 가장 많이 우승한 게임 TOP 5" 같은 글로벌 랭킹 제공.

## 3. 기술 스택

- **UI/UX 설계:** **Figma** (화면 레이아웃 및 대진표 애니메이션 흐름 정의)
- **DB 설계:** `ErdCloud` (User, WorldCup, Game, Response 간의 관계 정의)

| 분류                   | 기술 스택                    | 선택 이유                                        |
| -------------------- | ------------------------ | -------------------------------------------- |
| **Framework**        | **Next.js (App Router)** | 파일 기반 라우팅, SSR/SSG 지원, API Routes 통합         |
| **Language**         | **TypeScript**           | 게임 데이터(RAWG API)의 복잡한 타입을 안전하게 관리            |
| **Styling**          | **Tailwind CSS**         | 빠른 레이아웃 구성 및 대진표 반응형 디자인 최적화                 |
| **State Management** | **Zustand**              | 대진표 진행 상태(현재 라운드, 선택된 후보들)를 가볍게 관리           |
| **Data Fetching**    | **TanStack Query (v5)**  | 게임 검색 결과 캐싱 및 무한 스크롤 구현 용이                   |
| **Backend/DB**       | **Supabase**             | 별도 백엔드 구축 없이 DB, 인증(Auth), 이미지 스토리지를 한 번에 해결 |
테스트 및 검증 (Testing)

- **컴포넌트 테스트:** **Vitest** (대진표 알고리즘이 16강에서 8강으로 잘 넘어가는지 로직 검증)
- **E2E 테스트:** **Playwright** (실제 사용자가 게임을 검색하고 결승까지 마치는 전체 흐름 테스트)

배포 및 운영 (Deployment & Maintenance)

- **CI/CD & Hosting:** **Vercel** (Next.js를 만든 곳인 만큼 가장 완벽한 궁합과 자동 배포 지원)
- **Error Tracking:** **Sentry** (유저가 월드컵 도중 튕기거나 API 에러가 날 때 실시간 모니터링)

시스템 아키텍처 

- **Frontend (Next.js):** 유저가 게임을 검색하고 이상형 월드컵 진행 (Zustand로 상태 관리).
- **External API (RAWG):** 실시간으로 게임 이미지와 정보를 땡겨옴.
- **Backend (Next.js API Routes):** API Key 보안 관리 및 Supabase와 통신.
- **Database (Supabase):** 최종 우승작 데이터 및 설문 답변 저장.

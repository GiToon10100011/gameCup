## 1. 문서 개요

GameCup PRD를 기반으로 사용자 목표 중심의 유즈케이스를 정의한다.  
각 유즈케이스는 사용자와 시스템 간 상호작용을 단계 흐름으로 기술한다.  
UC-01~UC-04는 PRD Iteration 2/3 기반이며, UC-05~UC-10은 PRD Iteration 4 (v4.0) 기반으로 추가되었다.

---

## 2. 액터(Actor) 정의

|액터|설명|
|---|---|
|**일반 사용자**|비로그인 상태로 서비스를 이용하는 모든 방문자. MVP 범위의 주요 액터.|
|**인증 사용자**|이메일 매직 링크로 로그인하여 토너먼트 저장·이력·공유 기능을 이용하는 사용자. Iteration 4 주요 액터.|
|**비인증 방문자 (공유 링크)**|공개 결과 링크(F-20)를 통해 결과를 열람하는 비로그인 방문자.|
|**외부 게임 API**|게임 데이터(이름, 썸네일 등)를 제공하는 외부 시스템 액터 (RAWG API).|
|**Supabase Auth**|매직 링크 이메일 발송 및 인증 세션 관리를 담당하는 외부 인증 시스템 액터.|

## 4. 유즈케이스 상세 명세

### UC-01 · 게임 검색하기

**목표:** 사용자가 원하는 게임을 이름으로 검색하여 결과 목록을 확인한다.  
**관련 요구사항:** F-01, F-02, F-12, F-13, NF-01, NF-05

|단계|사용자|시스템|
|---|---|---|
|1|검색창에 게임 이름을 입력한다.||
|2||외부 게임 API에 검색 요청을 전송한다.|
|3||게임 썸네일과 이름을 포함한 결과 목록을 표시한다.|
|4|결과 목록을 확인한다.||

**예외 흐름**

- **1a.** 검색어가 비어 있거나 공백이면 → 시스템은 API를 호출하지 않고 목록을 표시하지 않는다. `(F-13)`
- **2a.** 동일 검색어의 이전 결과가 캐시에 존재하면 → 시스템은 API 호출 없이 캐시 결과를 표시한다. `(NF-05)`
- **3a.** API 호출이 실패하면 → 시스템은 오류 메시지를 표시하고 이전 상태를 유지한다. `(F-12)`

---

### UC-02 · 토너먼트 후보 구성하기

**목표:** 사용자가 1:1 대결에 참가할 게임 후보 목록을 원하는 구성으로 만든다.  
**관련 요구사항:** F-03, F-04, F-05

|단계|사용자|시스템|
|---|---|---|
|1|검색 결과에서 원하는 게임을 선택한다.||
|2||선택된 게임을 후보 목록에 추가하고 목록을 갱신하여 표시한다.|
|3|원하는 구성이 될 때까지 1~2를 반복한다.||
|4|후보 목록을 최종 확인한다.||

**예외 흐름**

- **2a.** 이미 후보 목록에 있는 게임을 선택하면 → 시스템은 중복 안내 메시지를 표시하고 추가하지 않는다. `(F-04)`
- **3a.** 사용자가 후보 목록에서 특정 게임의 삭제를 요청하면 → 시스템은 해당 게임을 목록에서 제거한다. `(F-05)`

---

### UC-03 · 토너먼트 진행하기

**목표:** 사용자가 1:1 대결 방식으로 게임들을 비교하여 최종 1개를 가린다.  
**관련 요구사항:** F-06, F-07, F-08, F-09, F-10, NF-02

|단계|사용자|시스템|
|---|---|---|
|1|토너먼트 시작을 요청한다.||
|2||후보 게임을 무작위 순서로 배치하여 첫 번째 라운드를 구성한다.|
|3||1:1 대결 화면을 표시한다.|
|4|두 게임 중 선호하는 게임을 선택한다.||
|5||선택된 게임을 다음 라운드 진출 목록에 등록한다.|
|6||라운드의 모든 대결이 끝나면 다음 라운드를 자동으로 구성하고, 3단계로 돌아간다.|
|7||최종 1개의 게임이 남으면 토너먼트를 종료한다.|

**예외 흐름**

- **1a.** 후보가 2개 미만이면 → 시스템은 시작 버튼을 비활성화하고 토너먼트를 시작할 수 없다. `(F-06)`
- **2a.** 라운드 구성 시 후보 수가 홀수이면 → 시스템은 마지막 게임을 부전승으로 처리하여 자동 진출시킨다. `(F-09)`

---

### UC-04 · 최종 결과 확인하기

**목표:** 사용자가 토너먼트 우승 게임을 결과 화면에서 확인하고, 이후 행동을 선택한다.  
**관련 요구사항:** F-10, F-11

|단계|사용자|시스템|
|---|---|---|
|1||우승 게임의 이름과 썸네일을 결과 화면에 표시한다.|
|2|결과를 확인한다.||
|3|새 토너먼트 시작 또는 처음 화면으로 이동을 선택한다.||
|4||선택에 따라 후보 목록과 진행 상태를 초기화하거나 초기 화면으로 이동한다. `[신규 도출]`|

---

### UC-05 · 매직 링크 로그인

**목표:** 사용자가 이메일 매직 링크를 통해 로그인하고 인증 세션을 수립한다.
**관련 요구사항:** F-14, F-15, NF-07

|단계|사용자|시스템|
|---|---|---|
|1|로그인 화면(AuthPage)에서 이메일 주소를 입력하고 "링크 전송"을 클릭한다.||
|2||AuthModule이 SupabaseClient를 통해 Supabase Auth에 매직 링크 발송을 요청한다.|
|3||"이메일을 확인하세요" 안내 메시지를 표시한다.|
|4|이메일 수신함에서 매직 링크를 클릭한다.||
|5||앱의 콜백 URL에서 onAuthStateChange 이벤트를 수신하고 세션(IUser)을 AuthSlice에 저장한다.|
|6||인증 완료 처리 후 토너먼트 허브(HubPage)로 리다이렉트한다.|

**예외 흐름**

- **1a.** 유효하지 않은 이메일 형식을 입력하면 → 시스템은 입력 오류를 안내하고 발송을 차단한다. `(F-14)`
- **4a.** 링크가 만료되었거나 콜백 오류가 발생하면 → 시스템은 오류 메시지를 표시하고 이메일 재입력 화면으로 돌아간다. `(F-14)`
- **4b.** 사용자가 메일을 수신하지 못한 경우 → 재발송 버튼을 통해 2단계부터 재시도할 수 있다. `(F-14)`
- **[인증 가드]** 비로그인 사용자가 보호 리소스에 접근하면 → 시스템이 접근을 차단하고 AuthPage로 리다이렉트한다. `(F-15)`

---

### UC-06 · 토너먼트(설문) 생성·저장

**목표:** 로그인 사용자가 게임 검색·후보 등록 과정을 거쳐 후보 세트에 이름을 붙여 토너먼트로 저장한다.
**관련 요구사항:** F-16, F-01~F-05 (재사용), F-15, NF-06

|단계|사용자|시스템|
|---|---|---|
|1|토너먼트 생성 화면(CreatePage)에서 게임 이름을 검색창에 입력한다.||
|2||외부 게임 API에 검색 요청을 전송하고 결과 목록을 표시한다. (UC-01 재사용)|
|3|검색 결과에서 원하는 게임을 선택한다.||
|4||선택된 게임을 후보 목록에 추가한다. (UC-02 재사용)|
|5|원하는 구성이 될 때까지 1~4를 반복한다.||
|6|토너먼트 이름을 입력하고 "저장" 버튼을 클릭한다.||
|7||TournamentStorageModule이 CandidateModule에서 후보 세트를 수집하고 SupabaseClient를 통해 저장한다.|
|8||저장 완료 후 TournamentLibrarySlice에 활성 토너먼트를 설정하고 HubPage로 이동한다.|

**예외 흐름**

- **6a.** 후보가 2개 미만이면 → 시스템은 저장 버튼을 비활성화하고 저장할 수 없다. `(F-16)`
- **6b.** 토너먼트 이름이 비어 있으면 → 시스템은 이름 입력을 요구하고 저장을 차단한다. `(F-16)`
- **7a.** Supabase 저장 중 오류가 발생하면 → 시스템은 오류 메시지를 표시하고 저장 이전 상태를 유지한다. `(F-16)`
- **[미인증]** 인증 세션이 없는 상태에서 저장을 시도하면 → 시스템이 AuthPage로 리다이렉트한다. `(F-15)`

---

### UC-07 · 내 토너먼트 선택·플레이 진입

**목표:** 로그인 사용자가 허브 화면의 내 토너먼트 목록에서 하나를 선택하여 플레이를 시작하거나, 필요 시 삭제한다.
**관련 요구사항:** F-17, F-15, NF-06

|단계|사용자|시스템|
|---|---|---|
|1||TournamentStorageModule이 SupabaseClient를 통해 사용자의 토너먼트 목록을 불러와 허브 화면에 표시한다.|
|2|목록에서 원하는 토너먼트를 선택한다.||
|3||선택된 토너먼트의 후보 세트를 불러오고 TournamentLibrarySlice에 활성 토너먼트를 설정한다.|
|4||PlayPage로 이동하여 토너먼트 진행을 시작한다. (UC-03으로 이어짐)|

**예외 흐름**

- **1a.** 목록 조회 중 네트워크 오류가 발생하면 → 시스템은 오류 메시지를 표시하고 재시도 버튼을 제공한다. `(F-17)`
- **[삭제]** 사용자가 토너먼트 삭제를 요청하면 → 시스템은 확인 요청 후 TournamentStorageModule.deleteTournament를 호출하여 해당 토너먼트를 삭제하고 목록을 갱신한다. `(F-17)`
- **[빈 목록]** 저장된 토너먼트가 0개이면 → 허브는 UC-08 온보딩 상태로 분기한다. `(F-17, F-18)`

### UC-08 · 온보딩 — 첫 토너먼트 생성

**목표:** 로그인 후 저장된 토너먼트가 없는 사용자에게 첫 토너먼트 생성을 안내하여 서비스에 자연스럽게 진입시킨다.
**관련 요구사항:** F-18, F-16 (연결), F-15

|단계|사용자|시스템|
|---|---|---|
|1||TournamentLibrarySlice에서 myTournaments.length === 0을 감지한다.|
|2||HubPage에 온보딩 안내 화면(빈 상태 메시지 + "첫 토너먼트 만들기" 버튼)을 표시한다.|
|3|"첫 토너먼트 만들기" 버튼을 클릭한다.||
|4||CreatePage로 이동하여 UC-06 흐름으로 연결한다.|

**예외 흐름**

- **3a.** 사용자가 온보딩 안내를 닫거나 이탈하면 → 시스템은 HubPage 빈 상태를 유지하며 강제 진행하지 않는다. `(F-18)`

---

### UC-09 · 결과 이력 저장·조회

**목표:** 토너먼트 플레이 완료 시 우승 게임과 결과를 이력으로 저장하고, 이후 해당 토너먼트의 과거 이력을 다시 조회한다.
**관련 요구사항:** F-19, F-15, NF-06

|단계|사용자|시스템|
|---|---|---|
|1||ResultModule.getWinner()를 통해 우승 게임을 결과 화면(ResultPage)에 표시한다.|
|2||TournamentStorageModule.saveResult()를 호출하여 우승 게임·플레이 시각·대진 요약(ITournamentResult)을 Supabase에 저장한다.|
|3||저장 완료 확인 메시지를 표시한다.|
|4|해당 토너먼트의 이력 조회를 요청한다.||
|5||TournamentStorageModule.listResults()를 호출하여 과거 이력 목록을 표시한다.|

**예외 흐름**

- **2a.** Supabase 저장 중 오류가 발생하면 → 시스템은 오류 메시지를 표시하되 결과 화면 자체는 유지한다. `(F-19)`
- **5a.** 이력이 없는 경우(첫 플레이) → 시스템은 "아직 이력이 없습니다" 메시지를 표시한다. `(F-19)`
- **[미인증]** 인증 세션이 없는 상태에서 저장을 시도하면 → 시스템이 저장을 건너뛰고 비로그인 결과 확인만 허용한다. `(F-15)`

---

### UC-10 · 결과 공유(링크)

**목표:** 인증 사용자가 완료된 토너먼트 결과를 공개 URL로 공유하고, 비로그인 방문자도 해당 링크로 결과를 열람한다.
**관련 요구사항:** F-20, F-15, NF-06

**[공유 생성 흐름]**

|단계|사용자|시스템|
|---|---|---|
|1|ResultPage에서 "공유 링크 생성" 버튼을 클릭한다.||
|2||TournamentStorageModule.createPublicShare(resultId)를 호출하여 IPublicShare(shareId)를 생성하고 Supabase에 저장한다.|
|3||생성된 공개 URL을 화면에 표시하고 복사 버튼을 제공한다.|
|4|URL을 복사하여 타인에게 공유한다.||

**[링크 열람 흐름]**

|단계|사용자|시스템|
|---|---|---|
|1|공개 URL에 접근한다.||
|2||TournamentStorageModule.getPublicResult(shareId)를 호출하여 결과 데이터를 조회한다.|
|3||PublicResultPage에 우승 게임 및 결과 정보를 표시한다.|

**예외 흐름**

- **2a.** [공유 생성] 공유 생성 중 오류가 발생하면 → 시스템은 오류 메시지를 표시한다. `(F-20)`
- **2b.** [링크 열람] 존재하지 않거나 만료된 shareId에 접근하면 → 시스템은 "결과를 찾을 수 없습니다" 메시지를 표시한다. `(F-20)`
- **[미인증 공유 생성 시도]** 인증 세션 없이 공유 생성을 시도하면 → 시스템이 로그인 안내 메시지를 표시한다. `(F-15)`

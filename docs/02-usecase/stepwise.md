## 1. 문서 개요

GameCup Iteration 2 PRD를 기반으로 사용자 목표 중심의 유즈케이스를 정의한다.  
각 유즈케이스는 사용자와 시스템 간 상호작용을 단계 흐름으로 기술하며,  
Iteration 3 PRD 개선의 근거 자료로 활용된다.

## 4. 유즈케이스 상세 명세

### UC-01 · 게임 검색하기

**목표:** 사용자가 원하는 게임을 이름으로 검색하여 결과 목록을 확인한다.  
**관련 요구사항:** F-01, F-02, F-12, F-13, NF-01, NF-05

|단계|사용자|시스템|
|---|---|---|
|1|검색창에 게임 이름을 입력한다.||
|2||외부 게임 API에 검색 요청을 전송한다.|
|3||게임 썸네일과 이름을 포함한 결과 목록을 표시한다.|
|4|결과 목록을 확인한다.||

**예외 흐름**

- **1a.** 검색어가 비어 있거나 공백이면 → 시스템은 API를 호출하지 않고 목록을 표시하지 않는다. `(F-13)`
- **2a.** 동일 검색어의 이전 결과가 캐시에 존재하면 → 시스템은 API 호출 없이 캐시 결과를 표시한다. `(NF-05)`
- **3a.** API 호출이 실패하면 → 시스템은 오류 메시지를 표시하고 이전 상태를 유지한다. `(F-12)`

---

### UC-02 · 토너먼트 후보 구성하기

**목표:** 사용자가 1:1 대결에 참가할 게임 후보 목록을 원하는 구성으로 만든다.  
**관련 요구사항:** F-03, F-04, F-05

|단계|사용자|시스템|
|---|---|---|
|1|검색 결과에서 원하는 게임을 선택한다.||
|2||선택된 게임을 후보 목록에 추가하고 목록을 갱신하여 표시한다.|
|3|원하는 구성이 될 때까지 1~2를 반복한다.||
|4|후보 목록을 최종 확인한다.||

**예외 흐름**

- **2a.** 이미 후보 목록에 있는 게임을 선택하면 → 시스템은 중복 안내 메시지를 표시하고 추가하지 않는다. `(F-04)`
- **3a.** 사용자가 후보 목록에서 특정 게임의 삭제를 요청하면 → 시스템은 해당 게임을 목록에서 제거한다. `(F-05)`

---

### UC-03 · 토너먼트 진행하기

**목표:** 사용자가 1:1 대결 방식으로 게임들을 비교하여 최종 1개를 가린다.  
**관련 요구사항:** F-06, F-07, F-08, F-09, F-10, NF-02

|단계|사용자|시스템|
|---|---|---|
|1|토너먼트 시작을 요청한다.||
|2||후보 게임을 무작위 순서로 배치하여 첫 번째 라운드를 구성한다.|
|3||1:1 대결 화면을 표시한다.|
|4|두 게임 중 선호하는 게임을 선택한다.||
|5||선택된 게임을 다음 라운드 진출 목록에 등록한다.|
|6||라운드의 모든 대결이 끝나면 다음 라운드를 자동으로 구성하고, 3단계로 돌아간다.|
|7||최종 1개의 게임이 남으면 토너먼트를 종료한다.|

**예외 흐름**

- **1a.** 후보가 2개 미만이면 → 시스템은 시작 버튼을 비활성화하고 토너먼트를 시작할 수 없다. `(F-06)`
- **2a.** 라운드 구성 시 후보 수가 홀수이면 → 시스템은 마지막 게임을 부전승으로 처리하여 자동 진출시킨다. `(F-09)`

---

### UC-04 · 최종 결과 확인하기

**목표:** 사용자가 토너먼트 우승 게임을 결과 화면에서 확인하고, 이후 행동을 선택한다.  
**관련 요구사항:** F-10, F-11

|단계|사용자|시스템|
|---|---|---|
|1||우승 게임의 이름과 썸네일을 결과 화면에 표시한다.|
|2|결과를 확인한다.||
|3|새 토너먼트 시작 또는 처음 화면으로 이동을 선택한다.||
|4||선택에 따라 후보 목록과 진행 상태를 초기화하거나 초기 화면으로 이동한다. `[신규 도출]`|

## 6. 유즈케이스 도출 중 발견된 PRD 문제점

|ISS ID|문제 유형|문제 상세|조치|
|---|---|---|---|
|ISS-01|사후 흐름 누락|F-11 결과 화면 이후 사용자 행동(재시작, 처음으로 이동)에 대한 요구사항이 없다.|Iteration 3에서 F-13(새 토너먼트 시작) 추가|

|ISS-03|F-07과 F-08 중복성|"1:1 대결 진행"과 "게임 선택"은 동일 UC 내 연속 단계로 분리할 실익이 적다. 과도한 세분화 우려.|Iteration 3에서 F-07에 F-08 통합, 설명 보강|

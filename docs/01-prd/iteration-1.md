# 1. 요구사항 정의서

## 1.1 문서 개요

GameCup Iteration 1(MVP) 범위의 기능 요구사항을 정의한다.  
게임 검색 → 후보 등록 → 1:1 토너먼트 진행까지의 핵심 플로우를 대상으로 한다.

## 1.2 프로젝트 배경

|항목|내용|
|---|---|
|**목표**|게임 취향 확인을 위한 인터랙티브 이상형 월드컵 서비스 구축|
|**타겟 사용자**|Steam 등 플랫폼 게이머, SNS에서 취향을 공유하는 커뮤니티 사용자|
|**핵심 가치**|API 자동완성으로 입력 편의 제공, 토너먼트 방식으로 참여율 향상|

## 1.3 기능 요구사항

| ID   | 기능명    | 상세 설명                                                           | 우선순위 |
| ---- | ------ | --------------------------------------------------------------- | :--: |
| F-01 | 게임 검색  | 검색창 텍스트 입력 시 RAWG API 호출 (디바운싱 300ms 적용). 게임 목록과 썸네일을 드롭다운으로 표시 |  필수  |
| F-02 | 후보 등록  | 검색 결과에서 게임 선택 시 후보 풀(Pool)에 추가. 중복 선택 시 알림 후 무시                 |  필수  |
| F-03 | 대진표 생성 | 후보 목록을 **Fisher-Yates 셔플** 후 1:1 대진표 구성. 최소 2개 이상 등록 시 활성화      |  필수  |
| F-04 | 월드컵 진행 | 1:1 대결에서 게임 선택 시 다음 라운드 배열로 진출. 최종 1개 남을 때까지 반복                 |  필수  |
⚠️ Fisher-Yates 셔플은 **모든 순열이 나타날 확률이 동일**하며, 배열을 단 한 번만 순회하므로 **시간 복잡도 $O(n)$** 의 최고 효율을 자랑한다. 일반 `sort()`는 특정 요소가 특정 위치에 배치될 확률이 높아지는 Bias(편향)이 발생할 수 있기에 Fisher-Yates셔플을 사용한다.

## 1.5 유즈케이스 명세

### UC-01 · 게임 검색 및 후보 등록

|**구분**|**내용**|
|---|---|
|**액터**|사용자|
|**사전 조건**|서비스 접속 완료|

**기본 흐름**

1. 사용자가 검색창에 게임 이름 입력
2. 시스템이 디바운싱(300ms) 후 RAWG API 호출
3. 드롭다운으로 게임 목록(썸네일 포함) 표시
4. 사용자가 게임 선택 → 후보 풀 추가
5. 검색창 초기화, 후보 목록 갱신

**예외/대안**

- 중복 게임 선택 시 경고 토스트 표시 후 무시
- API 오류 시 에러 메시지 표시

---

### UC-02 · 이상형 월드컵 진행

|**구분**|**내용**|
|---|---|
|**액터**|사용자|
|**사전 조건**|후보 게임 2개 이상 등록 완료|

**기본 흐름**

1. 사용자가 **시작** 버튼 클릭
2. 시스템이 후보 배열 셔플 후 1:1 매치업 렌더링
3. 사용자가 선호 게임 이미지 클릭
4. 선택된 게임이 다음 라운드 배열로 이동
5. 2개 이상 남으면 3번부터 반복, 1개면 우승 화면 표시

**예외/대안**

- 애니메이션 중 클릭 시 상태 잠금(Lock)으로 중복 입력 차단

# 2. 설계 문서

## 2.1 기술 스택

| 분류            | 기술                   | 선택 이유                      |
| ------------- | -------------------- | -------------------------- |
| **Framework** | Next.js (App Router) | SSR/SSG, API Route 통합 지원   |
| **언어**        | TypeScript           | 타입 안전성, 유지보수 편의            |
| **스타일**       | Tailwind CSS         | 유틸리티 클래스, 빠른 반응형 개발        |
| **상태 관리**     | Zustand              | 간결한 전역 상태, 보일러플레이트 최소화     |
| **API 캐싱**    | TanStack Query v5    | 서버 상태 캐싱, 중복 API 요청 방지     |
| **외부 API**    | RAWG API             | 50만+ 게임 메타데이터 및 고화질 썸네일 제공 |
| **테스트**       | Vitest               | 단위 테스트, 빠른 실행 속도           |

## 2.2 시스템 구성도

### 전체 흐름

```
사용자
  │
  ▼
SearchBar ──── useDebounce Hook
  │                  │
  │            RAWG API 호출
  │                  │
  ▼                  ▼
CandidatePool ◄── 검색 결과 (TanStack Query 캐싱)
  │
  │ [시작] 버튼
  ▼
TournamentBoard ◄──► tournamentStore (Zustand)
  │
  │ 1:1 선택 반복
  ▼
ResultScreen
```

### 폴더 구조 (핵심)

```
src/
├── app/
│   ├── page.tsx                  # 메인 화면 (검색 + 후보 등록)
│   └── tournament/
│       └── page.tsx              # 월드컵 진행 화면
├── components/
│   ├── SearchBar.tsx             # 검색창 + 드롭다운
│   ├── GameCard.tsx              # 게임 썸네일 + 이름 카드
│   ├── CandidatePool.tsx         # 후보 목록 표시
│   ├── TournamentBoard.tsx       # 1:1 대진표 렌더링
│   └── ResultScreen.tsx          # 최종 우승 결과 화면
├── store/
│   ├── candidateStore.ts         # 후보 게임 전역 상태
│   └── tournamentStore.ts        # 대진표·라운드 전역 상태
├── hooks/
│   └── useDebounce.ts            # 검색 디바운싱 커스텀 훅
└── lib/
    └── rawg.ts                   # RAWG API 호출 함수
```

## 2.3 컴포넌트 설계

|컴포넌트|역할|연결 대상|
|---|---|---|
|`SearchBar`|게임 이름 입력, 드롭다운 표시|`useDebounce` → RAWG API|
|`CandidatePool`|후보 게임 목록 표시 및 삭제|`candidateStore`|
|`TournamentBoard`|1:1 대진표 렌더링|`tournamentStore`|
|`GameCard`|게임 썸네일 + 이름 카드|`TournamentBoard`, `CandidatePool`|
|`ResultScreen`|우승 게임 최종 결과 화면|`tournamentStore.winner`|

## 2.4 상태 관리 설계 (Zustand)

|Store|주요 State|주요 Action|
|---|---|---|
|`candidateStore`|`candidates: Game[]`|`addCandidate`, `removeCandidate`|
|`tournamentStore`|`bracket`, `currentPair`, `round`, `winner`|`initTournament`, `selectWinner`, `nextRound`|

### tournamentStore 핵심 로직

```
① initTournament
   candidates 배열을 Fisher-Yates 셔플 → bracket[0]에 할당

② selectWinner
   선택된 게임을 winners 배열에 추가 → 다음 페어로 이동

③ nextRound
   현재 라운드 winners → 다음 bracket으로 설정
   winners가 1개면 → winner 상태 확정
```

## 2.5 RAWG API 연동

|항목|내용|
|---|---|
|**Endpoint**|`GET https://api.rawg.io/api/games`|
|**주요 파라미터**|`search={query}&key={API_KEY}&page_size=10`|
|**응답 필드**|`id`, `name`, `background_image`, `released`|
|**인증**|API 키 → `.env` (`NEXT_PUBLIC_RAWG_KEY`)|
|**호출 방식**|디바운싱 300ms 적용 후 TanStack Query로 캐싱|

## 2.6 주요 알고리즘

**셔플 및 대진표 생성**

- Fisher-Yates 알고리즘으로 후보 배열 무작위 셔플
- 셔플 결과를 2개씩 페어링하여 `bracket[round]` 배열 구성
- 홀수 후보 수일 경우 마지막 게임은 자동 부전승 처리

## 2.7 화면 플로우

```
1. 메인 화면
   └─ 게임 검색 → 후보 등록 (최소 2개)
       └─ [시작] 버튼 활성화

2. 토너먼트 화면
   └─ 대진표 생성 (셔플)
       └─ 1:1 선택 반복
           └─ 라운드 진행

3. 결과 화면
   └─ 최종 우승 게임 표시
```

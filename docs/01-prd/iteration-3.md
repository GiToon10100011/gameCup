## 3. 기능 요구사항

기능 요구사항은 **핵심 기능**으로 분류한다.  
Iteration 2의 선택 기능(F-12·F-13)이 핵심으로 상향됨에 따라, Iteration 3의 선택 기능은 없다.

### 3.1 핵심 기능 (Core)

> 서비스의 핵심 플로우를 구성하는 기능. Iteration 3 내 반드시 구현되어야 한다.

|ID|기능명|상세 설명|변경 사유|
|---|---|---|---|
|F-01|게임 검색|사용자는 게임 이름을 입력하여 외부 게임 데이터베이스에서 게임을 검색할 수 있어야 한다.|-|
|F-02|검색 결과 표시|검색 결과는 게임 썸네일 이미지와 이름을 포함한 목록으로 표시되어야 한다.|-|
|F-03|후보 등록|사용자는 검색 결과에서 게임을 선택하여 토너먼트 후보 목록에 추가할 수 있어야 한다.|-|
|F-04|중복 등록 방지|이미 후보 목록에 존재하는 게임을 다시 추가하려 할 경우, 사용자에게 알림을 표시하고 추가되지 않아야 한다.|-|
|F-05|후보 삭제|사용자는 후보 목록에서 원하는 게임을 제거할 수 있어야 한다.|-|
|F-06|토너먼트 시작|후보 게임이 2개 이상 등록된 경우, 사용자는 토너먼트를 시작할 수 있어야 한다. 후보가 2개 미만이면 시작이 불가능해야 한다.|-|
|F-07|1:1 대결 진행 및 게임 선택|토너먼트는 후보 게임을 무작위 순서로 배치하여 1:1 대결 방식으로 진행되어야 한다. 사용자는 대결에서 선호하는 게임을 선택할 수 있으며, 선택된 게임은 다음 라운드로 진출해야 한다.|기존 F-07+F-08 통합 (ISS-03)|
|F-08|라운드 자동 진행|한 라운드의 모든 대결이 완료되면, 승리한 게임들로 다음 라운드가 자동으로 구성되어야 한다.|기존 F-09 → F-08 재번호|
|F-09|부전승 처리|후보 게임의 수가 홀수일 경우, 대결 상대가 없는 마지막 게임은 자동으로 다음 라운드에 진출해야 한다.|기존 F-10 → F-09 재번호|
|F-10|결과 화면 표시|모든 라운드가 종료되어 최종 1개의 게임이 남으면, 해당 게임을 우승 게임으로 결과 화면에 표시해야 한다.|기존 F-11 → F-10 재번호|
|F-11|API 오류 안내|외부 게임 데이터 조회에 실패한 경우, 사용자에게 오류 메시지를 표시하고 서비스는 정상 동작을 유지해야 한다.|선택 → 핵심 상향 (ISS-02)|
|F-12|빈 검색어 처리|검색어가 공백이거나 비어 있는 경우, 외부 API를 호출하지 않고 드롭다운을 표시하지 않아야 한다.|선택 → 핵심 상향 (ISS-02)|
|F-13|새 토너먼트 시작|결과 화면에서 사용자는 후보 목록과 진행 상태를 초기화하고 새 토너먼트를 시작할 수 있어야 한다.|신규 추가 (ISS-01)|

## 4. 비기능 요구사항

> 시스템의 품질 속성을 정의한다. 특정 구현 방법이 아닌, 시스템이 만족해야 하는 조건을 기술한다.

|ID|분류|요구사항|변경 사유|
|---|---|---|---|
|NF-01|**응답성**|검색 입력 후 1초 이내에 결과 목록이 화면에 표시되어야 한다.|-|
|NF-02|**안정성**|짧은 시간 내 연속 클릭 등 비정상적인 입력이 발생하더라도 데이터 불일치 없이 정상 상태를 유지해야 한다.|-|
|NF-03|**브라우저 호환성**|Chrome, Edge, Firefox의 최신 안정 버전에서 동일하게 동작해야 한다.|-|
|NF-04|**확장성**|향후 기능(결과 저장, 공유, 랭킹 등)을 추가할 수 있도록, 상태 관리 및 화면 구조가 기능 단위로 분리되어야 한다.|-|
|NF-05|**불필요한 외부 호출 최소화**|동일한 검색어에 대한 외부 API 응답은 세션 내에서 재활용되어야 하며, 중복 호출이 발생하지 않아야 한다.|표현 명확화 (ISS-04)|
